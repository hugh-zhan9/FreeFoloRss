# 移除全局 AI 聊天命令与快捷键设计

**目标**：彻底删除“全局 AI 聊天”命令、快捷键与设置入口，保证应用启动后不再注册或响应该命令，也不再展示任何相关配置。

## 背景与约束

- 需求明确要求“彻底删除”，不是隐藏或软禁用。
- 不引入 UI 配置；数据库相关配置仍由 .env 控制（本次不改 DB 逻辑）。
- 需清理所有引用，避免幽灵命令或热键残留。

## 方案对比

1. 彻底删除（选定）：删除命令 ID、注册、热键绑定、设置入口与所有引用。
2. 软删除：保留命令常量但不注册（不符合“彻底删除”）。
3. 特性开关禁用：保留代码但用配置禁用（不符合“彻底删除”）。

## 设计与范围

### 涉及模块

- 命令系统：命令 ID、命令列表、命令注册。
- 热键系统：默认绑定与监听注册。
- 设置页：移除“设置-快捷键-全局 AI 聊天设置”入口。

### 数据流/行为

- 启动时不再注册该命令。
- 热键系统不会绑定或监听该命令。
- 设置页不再展示相关条目。

### 错误处理

- 若历史配置或快捷键记录仍存在：启动后直接忽略（命令不存在即无效）。

## 测试与验收

- 命令绑定测试：命令列表不包含该命令。
- 热键绑定测试：快捷键映射中不存在该命令。
- UI 验收：设置页不存在“全局 AI 聊天设置”入口，热键不再触发 AI 聊天。
